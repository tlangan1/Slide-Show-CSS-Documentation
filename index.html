Miscellaneous Notes

  • Placing a style element with the contenteditable attribute set to true in an html page as follows: 
    <style contenteditable="true">
    ...
</style>
    in combination with the CSS rule that follows: 
    style[contenteditable="true"] {
    display: block;
}
    provides a way to see and edit CSS directly in the browser. You don’t have to open the browser tools to do so or change the code in your IDE and refresh the page. This is the technique used throughout this page.

  • In viewing the course, I ran into a concept known as the shadow DOM. This is not at all new and is used extensively in custom elements. It allows the author to attach a DOM tree to a node in the DOM that does not interact with the rest of the DOM in any way.
  • If you position something absolutely it is positioned relative to the first ancestor that is positioned. So, in practice, it is often the case that the parent of something positioned absolutely is positioned:relative.

Basic Selectors, relational selectors
  • The three most common selectors are targeting by id, the hash # sign, targeting by class, the “.” and targeting by element name, for example p for paragraph.
  • Descendent selector: For example, ul li targets all lis nested within a ul regardless of how far down the nesting may be. The space “ “ between the elements denotes this selector.
  • Child selector: For example, ol > li, targets all lis that are direct children of an ol. The greater than sign “>” between the elements denotes this selector.
  • Adjacent sibling: For example, li.myClass + li, targets any li immediately following in the mark-up of an li of class myClass. The plus sign “+” between the elements denotes this selector.
  • General sibling selector: For example, li.myClass ~ li targets all lis following in the mark up an li of class myClass. The tilde “~” between the elements denotes this selector.

Attribute selectors: let element = E and attribute = A
  • Simple attribute: For example, E[A] will target all E elements containing A attributes.

  • Attribute value: For exampler, E[A=”val”] will target all E elements containing A attributes that have the value “val”. In elements where the attribute value is case sensitive then the CSS selector is case sensitive otherwise not. This is denoted by the “=”.

    In this example this li has a class attribute with a value of 'the-target-attribute'

  • Note that the syntax “^=”, starts with, “$=” ends with and “*=” matches or contains are identical syntax to regular expressions.
  • Note that multiple attributes work like li[class^=c][role*=ol].

Pseudo-elements General Understaing

  • The pseudo-elements "::before" and "::after" are used to generate content and although you may visually see that generated content it is not part of the DOM; however, you still can style it. The only required attribute in pseudo-selectors is “content”, even if it is empty.
  • The way to think of the placement of these pseudo elements is before and after the contents of the element they are targeting, not before and after the element itself. For example, the pseudo elements p::before and p::after should be thought of as existing within the p as illustrated below: 
                    
<p>
    <::before>before content</::before>
    paragraph contents
    <::after>after content</::after>
</p>
  • Self-closing elements, such as img, cannot be targeted by these pseudo-elements. This makes sense since the pseudo-elements are created as children to the targeted element but a self-closing element cannot have children.

One common use case for a media query might be to prevent some sections of a web page from being included in the printed version of the page. The example below would prevent any content of class 'header' or 'sidebar' from being included. 

@media print {
    .header {display none;}
    .sidebar {display: none;}
}

Media Queries #2

  • The typical use of media queries is for responsive design and the most commonly used comparators or min-width, max-width.
  • @orientation is a comparator that could be helpful on a smartphone, for example, to change the layout depending on which way the user has the phone tipped.
  • Comparators can be combined operators like “and” or with the comma “,” which means “or”. The “not” and “only” operators are also used but I think the “only” is there just to prevent older browsers from misinterpreting media queries.
  • CSS @supports go into your CSS like @media queries do. @supports allows developers to check style support in a number of different ways so that they can more easily provide the correct styling in a given environment. Consider the following example that checks to see whether an environment supports “flex” before styling all divs as flex divs. 
    @supports (display: grid) {
    div {display: grid;}
{
    The JavaScript counterpart to @supports is CSS.supports.

Colors

  • Color formats in css for white are:
    • White
    • #fff
    • #FFFFFF
    • rgb(255, 255, 255)
    • rgb(100%, 100%, 100%)
    • rgba(255, 255, 255, 1)
    • rgba(100%, 100%, 100%, 1)
    • hsl(0, 100%, 100%)
    • hsla(0, 100%, 100%, 1)
  • hsl stands for hue, saturation, lightness and the a in hsla stands for alpha channel.
    • Hue: is the actual and is designated by an angle on the color circle which can be represented using various units.
    • Saturation: This is how strongly the given hue is represented with 100% being pure color and 0% being grey.
    • Lightness: This is how light the given hue is represented with 100% being white, 50% being normal and 0% being black.
  • The keyword “transparent” is actually a shortcut for rgba(0, 0, 0, 0). Hence. It is transparent black.

Fonts and Box Shadows

@font-face defines the name of a font-family and the source files that contain the characters as well as the font-weight, font-style and charset. This is another example of a CSS at rule. See MDN for detailed information.

The terms font and type face are often used interchangeably but, strictly speaking, they are not the same thing. For example, Helvetica is a typeface whereas Helvetica Regular 9 point is a specific font.

See MDN for the box-shadow specification. There are two interesting features. If you use the keyword “inset” the shadow is placed inside the border as if it is depressed in the page instead of the default appearance of a drop shadow giving the appearance of hovering over the page. The second feature is the spread radius. This value indicates how much of the shadow is solid whereas the blur-radius indicates the blurred portion of the shadow.

Text Decorations & Text Shadows

The syntax for text shadow is “text-shadow: offset-x offset-y blur radius color” and a decent set of text-shadow values are “text-shadow: 2px 5px 1px rgba(0, 0, 0, .2)”. Notice the use of the .2 for the alpha channel transparency in the black…that is the key. It is applied to this content.

In fact, multiple text shadows can be applied to the same text. See this content as an example. Also, notice the text-decoration here and how it is accomplished. There are a ton of text decoration options.

Border and Backgrounds #4: Background Position

This property is straight forward except when using percentages for left, right, top and bottom. When using percentages the point within the image that is used as the target to make the move is actually the point within the image at the same percentages. For example, if you positioned a background image right 100% and bottom 100% the point in the background image to which this calculation is being made would be the bottom right corner. If, on the other hand you position the background image right 50% bottom 50% then the center point of the image is actually the point within the image that is being positioned.
